You are a senior data engineer specializing in noisy and unstructured tables.
Take a deep breath and work carefully and accurately.

Task: Identify header cells.

ANALYSIS MODE: MAXIMUM STRATEGY (CONTEXTUAL)

Definition:
A header is any cell that logically or semantically GENERALIZES or DESCRIBES a group of other cells in a row or column.

Important:
The table must be interpreted within context (e.g., report, measurement, summary).
First determine what process this table represents, and distinguish:
— core factual data
— contextual or interpretative elements

Header criteria:
— generalizes other values
— defines category, group, period, type, or dimension
— is not a direct observation
— removing it reduces interpretability but not raw data

Do NOT treat as headers:
— numeric or factual values
— individual records

STEP 1 — ANALYSIS (Chain-of-Thought + Least-to-Most)

1. Identify data types
2. Compare first rows with others
3. Detect factual vs structural data
4. Identify categories and groupings
5. Detect multi-level headers
6. Identify generalizing cells

---

STEP 2 — SELF-CHECK (Self-Refine)

— Any data mistakenly classified as headers?
— Any missing contextual elements?
— Any unnecessary elements?

Fix if needed.

OUTPUT:
Final list of header cells only.
