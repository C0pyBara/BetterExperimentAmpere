You are a senior data engineer specializing in complex and noisy tables.

Task: Identify header cells using the MAXIMUM STRATEGY (CONTEXTUAL).

Definition:
A header is any cell that logically or semantically GENERALIZES or DESCRIBES a group of other cells in a row or column.

Important:
Interpret the table in its domain context (medical, statistical, economic, etc.).
First determine what this table represents, then distinguish:
  — core factual data (measurements, observations, values)
  — contextual elements (labels, categories, section names, dimension names)

Header criteria (include if ANY apply):
  — generalizes or names a group of values
  — defines a category, group, period, type, or dimension
  — is not a direct observation or measurement
  — removing it reduces interpretability of surrounding cells
  — acts as a section separator or group label within the table body
  — spans multiple columns or rows to label a broader group

Do NOT treat as headers:
  — numeric or factual values
  — individual records or observations
  — data cells that happen to appear in row 0

--- STEP 1 — ANALYSIS (Chain-of-Thought)
1. What domain/process does this table represent?
2. Scan row 0 and row 1: are they column labels or data?
3. Scan column 0: does it contain row labels or data values?
4. Are there mid-table rows that act as section separators?
5. Are there spanning cells that group multiple columns or rows?
6. What is the minimum context needed to interpret any data cell?

--- STEP 2 — SELF-CHECK (Self-Refine)
  — Did I include any raw data values as headers?
  — Did I miss any section labels or group separators?
  — Did I include all levels of a multi-level header?
  — Are spanning cells represented by their anchor (top-left) coordinate?
