You are a data engineer specializing in relational data modeling.
Take a deep breath and work carefully.

Task: Identify which cells in the table are headers.

ANALYSIS MODE: MINIMUM STRATEGY (STRUCTURAL)

Definition:
A header NAMES an attribute or dimension. It answers "what is this column/row measuring?"
A non-header IS data — a value, measurement, entity key, or category label.

Simple test per cell: "Does this cell name an attribute, or is it a value of an attribute?"

INCLUDE: column attribute names, row dimension labels, sub-column labels, group names.
EXCLUDE: row-level entity keys (values of the row dimension), aggregates, section separators.

--- STEP 1 — ANALYSIS
1. What does each COLUMN measure? The cells that name what a column measures are headers.
2. What does each ROW identify? The cell naming the ROW AXIS is a header (e.g. "Variable", "Country").
   BUT the actual row identifiers in that column (e.g. "Age", "Germany") are VALUES, not headers.
3. Are there multi-level column headers? Both levels name attributes → both are headers.
4. Are there section separators mid-table? → EXCLUDE (they label groups, not attributes).
5. Are there aggregate rows (Total, Mean, Sum)? → EXCLUDE.
6. Are there empty corner cells? → EXCLUDE.

--- STEP 2 — SELF-CHECK (Self-Refine)
  — Did I include any row-level entity identifiers? (values of the row dimension) → remove
  — Did I include any section separator rows? → remove
  — Did I include any aggregates or totals? → remove
  — Can I reconstruct the column schema from only the cells I included? → if yes, done
Correct if needed.

--- OUTPUT
List only header cell coordinates with cell text. No reasoning, no explanation.

OUTPUT FORMAT: one header per line: row col | cell text
Use 0-based indexing. For spanning cells report the top-left anchor coordinate.
