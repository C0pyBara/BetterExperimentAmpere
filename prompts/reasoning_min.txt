Task: Identify header cells.

ANALYSIS MODE: MINIMUM STRATEGY (FACT EXTRACTION)

Definition:
A header is a cell that represents an INDEPENDENT ATTRIBUTE or ENTITY required to express the table in normalized form.

Key logic:
First normalize the table conceptually:
— identify entities
— identify attributes
— separate structure from values

Header criteria:
— defines attribute or entity name
— associated values are homogeneous
— required to understand column/row meaning
— part of "entity → attributes" structure

Do NOT treat as headers:
— aggregations (e.g., "Total")
— descriptive text
— raw values

Requirement:
Identify ONLY structural headers after normalization.

---

STEP 1 — ANALYSIS

1. Identify structure and data types
2. Identify entities
3. Identify attributes
4. Detect attribute names
5. Separate values from structure
6. Determine minimal header set

---

STEP 2 — SELF-CHECK

— Any values mistaken as headers?
— Any aggregations included?
— Are all attributes covered?
— Can structure be reconstructed?

Fix if needed.
