Task: Identify header cells using the MINIMUM STRATEGY (STRUCTURAL / NORMALIZATION).

Definition:
A header is a cell that represents an INDEPENDENT ATTRIBUTE or ENTITY NAME required to express the table in normalized (relational) form.

Key logic:
Mentally normalize the table into flat relational tuples:
  — identify entities (what each row describes)
  — identify attributes (what each column measures)
  — separate structural labels from data values

Header criteria (include ONLY if ALL apply):
  — names an attribute or entity dimension
  — its associated values are homogeneous (same type across the group)
  — required to reconstruct the relational structure of the table
  — part of an "entity → attribute → value" triplet

Do NOT treat as headers:
  — aggregations or totals ("Total", "Sum", "Average", "Subtotal")
  — descriptive or explanatory cells mid-table that are not attribute names
  — raw values, even if they contextually label a group
  — section separators unless they define a new relational entity
  — data cells that appear in the first row

--- STEP 1 — ANALYSIS (Chain-of-Thought)
1. What entities does this table describe?
2. What attributes (column names) uniquely identify each measurement?
3. Are there row identifiers that serve as entity keys?
4. Which cells are attribute names vs data values?
5. Can the table be expressed as (entity, attribute, value) triples?
6. What is the minimal set of cells required for that structure?

--- STEP 2 — SELF-CHECK (Self-Refine)
  — Did I include any aggregate rows (totals, averages)?
  — Did I include any pure data values as headers?
  — Can the table structure be fully reconstructed from only these headers?
  — Am I including more than the minimal structural set?
